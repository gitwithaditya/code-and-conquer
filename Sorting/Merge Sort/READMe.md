# 🔀 Merge Sort

*Divide the array, sort each half, and merge them back together.*

</div>

---

## 📖 Overview

Merge Sort follows the **Divide and Conquer** approach. It recursively splits the array into smaller parts, sorts them, and then merges them into a sorted array.

It is one of the most efficient and reliable sorting algorithms.

---

## ⚡ Complexity

| Best | Average | Worst | Space |
|:---:|:---:|:---:|:---:|
| O(n log n) | O(n log n) | O(n log n) | O(n) |

**Stable:** ✅ Yes  
**In-place:** ❌ No

---

## ✅ Pros

- Guaranteed **O(n log n)** performance
- Stable sorting algorithm
- Efficient for large datasets

---

## ❌ Cons

- Requires extra memory
- Recursive implementation can be harder to understand

---

## 💡 Key Takeaway

> Merge Sort guarantees **O(n log n)** time complexity, making it a reliable choice for sorting large datasets where stability is important.
