# 📖 What is Sorting?

Sorting is the process of arranging elements in a specific order, usually:

- 🔼 Ascending Order
- 🔽 Descending Order

It is one of the most fundamental concepts in Data Structures & Algorithms and is widely used to optimize searching, data processing, and many advanced algorithms.

---

# 🤔 Why is Sorting Important?

Sorting makes many problems easier and more efficient.

Some common applications include:

- 🔍 Binary Search
- 📊 Data Analysis
- 🔄 Removing Duplicates
- 📂 Database Management
- 📈 Ranking Systems
- 🧩 Greedy Algorithms
- 🌳 Trees & Graph Algorithms

Many interview problems become much easier after sorting the input first.

# 📚 Types of Sorting Algorithms

## 🔹 Comparison-Based Sorting

These algorithms compare elements to determine their order.

Examples:

- Bubble Sort
- Selection Sort
- Insertion Sort
- Merge Sort
- Quick Sort

---

## 🔹 Non-Comparison Sorting

These algorithms do not compare elements directly.

Examples:

- Counting Sort
- Radix Sort
- Bucket Sort

These can achieve linear time under certain conditions.

---

# ⚖️ Stable vs Unstable Sorting

## ✅ Stable Sorting

Maintains the relative order of equal elements.

Examples:

- Bubble Sort
- Insertion Sort
- Merge Sort

---

## ❌ Unstable Sorting

May change the relative order of equal elements.

Examples:

- Selection Sort
- Quick Sort

---

# 💾 In-place vs Out-of-place Sorting

## ✅ In-place Sorting

Uses little or no extra memory.

Examples:

- Bubble Sort
- Selection Sort
- Insertion Sort
- Quick Sort

---

## ❌ Out-of-place Sorting

Requires additional memory.

Example:

- Merge Sort

---

# 🎯 Choosing the Right Sorting Algorithm

| Situation | Recommended Algorithm |
|-----------|----------------------|
| Small Array | Bubble / Insertion Sort |
| Nearly Sorted Array | Insertion Sort |
| Large Dataset | Merge Sort |
| General Purpose | Quick Sort |
| Stable Sorting Required | Merge Sort |
| Learning Fundamentals | Bubble Sort |

# 💡 Interview Tips

- Always begin with the brute-force approach.
- Understand why a sorting algorithm works before memorizing it.
- Learn the time and space complexity of each algorithm.
- Know whether the algorithm is Stable or Unstable.
- Know whether it is In-place or Out-of-place.
- Be able to explain the intuition behind your implementation.
